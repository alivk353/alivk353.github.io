#### JackSon序列化

JackSon会在序列化属性时会优先调⽤此属性的get⽅法，随后才会判断权限后通过反射取属性值:

⾸先会判断对象中的public、private、protected、final作用域属性是否包含getXxx⽅法，如果有则通过此⽅法获取属性

没有则判断该属性是否可访问（public），如果可访问则通过反射的⽅式取此值，反之则跳过此属性

> JackSon不会序列化静态属性

#### Jackson反序列化

反序列化时会调⽤对应属性的set⽅法，如果不存在此⽅法，则通过反射赋值(public)

反序列化时要求类内存在⼀个⽆参构造⽅法，在反序列化时会调⽤此⽅法创建对象

### Jackson对Java多态的⽀持

⼀个最简单的多态模型, 属性中所标记的类名，不⼀定就是其实例化的类，有可能标记的是⽗类，是接⼝, 此时正常情况下Jackson则⽆法正常反序列

Jackson也是⽀持这种多态的序列化与反序列化, 需要开启了 **Default Typing** 机制:

`objectMapper.enableDefaultTyping();`

开启了Default Typing，则如果序列化的对象属性中有对象，则该属性值是⼀个数组，索引为[]0表明该对象所表示的类，索引[]1表示该对象的属性以及属性值

```json
{"name":"name","sex":"sex","className":"classNmae","classType":["classType.className",{"key":"key","value":"value"}]}
```

序列化和反序列化，如果属性为对象，则会继续调⽤此对象的get和set⽅法为此对象属性赋值，与正常情况⼀样

调用`objectMapper.readValue(json, cls)` 一般情况json由用户控制, cls不可控由业务决定

如果cls=Object.class 代表可以反序列成任何类

再开启`objectMapper.enableDefaultTyping();` 反序列化下面的TemplatesImpl会调用getOutpropertities() 触发字节码加载

```json
[
 "com.sun.org.apache.xalan.internal.xsltc.trax.TemplatesImpl",
 {
 "transletBytecodes": ["class_byte[]"],
 "transletName": "a.b",
 "outputProperties": {}
 }
]
```

JackSon在还原对象属性时的逻辑, 调用getter的逻辑:

先定义一个类似TemplatesImpl结构的Info类:

- 存在getter方法`getProperties`
- 类没有定义对应的Field properties
- 返回值是Collects集合类的子类 Properties

```java
    public static class Info {
        String name ;
        public Info(String name) { this.name = name; }
        public Info() {}
        public String getName() { return name; }
        public void setName(String name) { this.name = name; }
        public Properties getProperties() { return new Properties(); }
    }
```

```json
["com.example.Info", {"name":"name", "properties":{"key":"vavvlaue"}}]
```

Jackson默认内置的黑名单:**com.fasterxml.jackson.databind.jsontype.impl.SubTypeValidator** 比如:

```java
s.add("com.sun.org.apache.xalan.internal.xsltc.trax.TemplatesImpl");
s.add("org.apache.xalan.xsltc.trax.TemplatesImpl");
```

> 在 Jackson 2.10 之后，enableDefaultTyping() 已经被标记为过期（Deprecated），取而代之的是更安全的 activateDefaultTyping()，要求开发者必须显式指定基础类型检查器（PolymorphicTypeValidator）
